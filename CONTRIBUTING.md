# Contributing to VaultShip

Thanks for contributing to VaultShip.

## Development Setup

```bash
cargo build --workspace
cargo test --workspace
```

## Pull Request Guidelines

- Keep PRs focused and reviewable.
- Add tests for new behavior or bug fixes.
- Update docs when command behavior or config changes.
- Ensure `cargo fmt`, `cargo clippy`, and `cargo test` pass.

## Commit Style

Use clear, imperative messages, for example:

- `add bind file signature validation`
- `fix registry auth header handling`

## Reporting Issues

For bugs and feature requests, open a GitHub Issue in:

- `https://github.com/cyberxdefend/vaultship`
